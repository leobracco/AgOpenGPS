La fila resaltada es donde está parado el nodo ahora mismo. Cada columna es una capa que tiene que estar arriba para que la siguiente funcione. Si el broker MQTT está caído del lado de la PC, todos los nodos se ven igual aunque estén vivos en LAN.

WiFi MQTT Target llega al nodo Status llega a PC Estado real Cómo se ve / qué hacer
— ✗ broker off — — Broker MQTT caído en la PC Verificar AgIO/CoreX en la PC. Reconectar desde Nodos → Diagnóstico MQTT.
✗ off — — — Apagado / fuera de LAN El nodo no responde. Anuncio retenido viejo. Chequear alimentación, antena, o si quedó en AP fallback (SSID propio visible desde celu).
✓ ✗ — — WiFi sí, broker no Verificar IP del broker en el nodo (portal web del nodo → broker). Si está bien, reiniciar nodo.
✓ ✓ ✗ ✓ Nodo vivo, bridge dormido El nodo está sano. PilotX no está mandando target (motor desactivado, sección sin asignar, o bridge sin Start). Revisar la página del producto.
✓ ✓ ✓ ✗ Recibe, no responde Status > 3 s atrás. Watchdog HW, colisión de ClientID, o nodo trabado. Reiniciar el nodo.
✓ ✓ ✓ ✓ OK pleno El nodo está respondiendo y recibiendo órdenes. Telemetría se actualiza ~10 Hz.
